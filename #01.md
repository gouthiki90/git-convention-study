# Git Commit Message Convention이란 무엇인가

> Git 커밋 메시지 컨벤션(Git Commit Message Convention)

프로젝트 참여자들이 일관된 커밋 메시지를 작성하기 위한 규칙이다.
관습적으로 통용되는 가이드라인도 있지만, 각 프로젝트에 따라서 만들어지는 규칙도 분명히 있다.



# Git Commit Message Convention을 사용해야 하는 이유는 무엇인가?

> 프로젝트에 있어 Git의 규칙은 협업에서 가장 중요한 부분 중 하나이다.

정해진 규칙에 따라 일관성 있는 커밋 메시지를 사용하면 서로의 작업물에 대한 이해도가 높아진다.
또한 협업에 있어 효율적으로 대처할 수 있다.

나도 실무에서 협업을 진행할 때, 커밋 메시지를 정확히 적어두지 않는 커밋이 있다면 코드에 주석이 없는 것처럼 이 작업물이 어떤 작업물이고 어떻게 수정해야 될지 막막해진다.
따라서 이렇게 정리해 보았다.

- 커밋 메시지의 가독성
- 디버깅 중 커뮤니케이션의 효율성
- 커밋 이력 추적
- 버그 또는 이슈 해결 속도 향상
